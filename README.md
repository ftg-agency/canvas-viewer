# Canvas Viewer

Простой React-вьюер для нескольких Obsidian `.canvas` файлов (стандарт [JSON Canvas](https://jsoncanvas.org)).
Pan / zoom / scroll, переключение между файлами и **запоминание последней позиции для каждого файла отдельно**.

## Возможности

- Рендер как в Obsidian: markdown в текстовых нодах, группы, цвета, стрелки выходят из правильных сторон.
- File-ноды: картинки инлайн, PDF в `<iframe>`, `.md` рендерятся инлайн, остальное — карточка-ссылка.
- Link-ноды: карточка URL с favicon.
- Позиция (pan + zoom) сохраняется в `localStorage` по каждому файлу — файл открывается там, где ты его оставил.
- Файлы подхватываются автоматически: положил `.canvas` в `canvases/` → пересборка → он в сайдбаре (правок кода не нужно).

## Локальный запуск

```bash
npm install
npm run dev
```

Открой адрес из консоли Vite (обычно http://localhost:5173).

## Где лежат канвасы

Всё кладётся в папку **`canvases/`** в корне репозитория — синхронизированный срез vault'а
с сохранением относительных путей:

```
canvases/
├─ demo.canvas
├─ roadmap.canvas
├─ Attachments/diagram.svg   # картинки/PDF, на которые ссылаются file-ноды
└─ Notes/readme.md           # .md, встроенные через file-ноды
```

`.canvas` и `.md` инлайнятся в бандл (`?raw`), картинки/PDF становятся хешированными
ассетами (`?url`) — base-path для GitHub Pages учитывается автоматически.

## Деплой на GitHub Pages (CI/CD)

1. Создай репозиторий на GitHub, запушь этот код в ветку `main`.
2. В репозитории: **Settings → Pages → Build and deployment → Source = GitHub Actions**.
3. Каждый push в `main` собирает и публикует сайт через `.github/workflows/deploy.yml`.
   Base-path берётся из имени репозитория автоматически (`VITE_BASE=/<repo>/`) — ничего править не надо.

Сайт будет доступен по адресу `https://<user>.github.io/<repo>/`.

> Если деплоишь не на project-сайт, а на корневой `https://<user>.github.io/`
> (репозиторий `<user>.github.io`), убери `env: VITE_BASE` из workflow — base останется `/`.

## Управление

- **Перемещение:** перетаскивание по пустому фону.
- **Зум:** колесо мыши / pinch на трекпаде, либо кнопки `+` / `−` слева внизу.
- Снизу слева — контролы и мини-карта.

Хочешь трекпад-режим (двумя пальцами — пан, pinch — зум, как в Figma)?
В `src/components/CanvasViewer.jsx` поставь `panOnScroll` вместо `zoomOnScroll`.

## Доработки на потом

- `[[wikilinks]]` / `==highlights==` как кликабельная навигация между канвасами.
- Deep-link на файл (`?file=foo`).
- Ленивая загрузка канвасов (`import.meta.glob` с `eager: false`), если их станут десятки.
